# 1장 코틀린이란 무엇이며, 왜 필요한가?

## 1.1 코틀린 맛보기
~~~
    1. 리스트에서 가장 나이 많은 사람을 찾기 위해 maxBy 함수를 사용한다.
    2. maxBy 함수에 전달한 람다식은 파라미터를 하나 받는다.
    3. it(별도로 파라미터 이름을 정의하지 않아도) 이라는 이름을 사용하면 람다 식의 유일한 인자를 사용할 수 있다.
    4. 엘비스 연산자라고 부르는 ?:는 null인 경우 0을 반환하고 그렇지 않은 경우 값을 반환한다.
~~~

## 1.2 코틀린의 주요 특성
~~~
    1. 자바가 사용되고 있는 모든 용도에 적합하면서도 더 간결하고 생산적이며 안전한 대체 언어를 제공하고있다. 
    2. 코틀린은 구체적인 영역의 문제를 해결하거나 특정 프로그래밍 패러다임을 지원하는 여러 라이브러리와 아주 잘 융합된다.
    3. 자바와 마찬가지로 코틀린도 정적 타입 지정언어 이다.
    4. 정적 타입 지정이라는 말은 모든 프로그램 구성 요소의 타입을 컴파일 시점에서 알 수 있고 
       프로그램 안에서 객체의 필드나 메서드를 사용할 때마다 컴파일러가 타입을 검증해준다는 뜻이다. 
    5. 코틀린은 자동으로 컴파일러가 문맥을 고려해 변수 타입을 결정하는 이런 기능을 타입추론이라고 한다.
    
    
    - 함수형 프로그래밍
    
    1. 일급시민인 함수 : 함수를 일반 값처럼 다룰 수 있다. 함수를 변수에 저장할 수 있고, 
                        함수를 인자로 다른 함수에 전달할 수 있으며, 함수에서 새로운 함수를 만들어서 반환할수 있다.
    2. 불변성 : 함수형 프로그래밍에서는 일단 만들어지고 나면 내부 상태가 절대로 바뀌지 않는 불변 객체를 사용해 프로그램을 작성한다.
    3. 부수 효과 없음 : 함수형 프로그래밍에서는 입력이 같으면 항상 같은 출력을 내놓고 다른 객체의 상태를 변경하지 않으며,
                       함수 외부나 다른 바깥 환경과 상호작용하지 않는 순수 함수를 사용한다.
                 
                 
    - 함수형 프로그래밍을 사용하면 유익한점
    
    1. 간결성
    2. 다중 스레드를 사용해도 안전하다.
    3. 함수형 프로그래밍은 테스트가 쉽다.
~~~

## 1.3 코틀린의 철학
~~~
    - 실용성
    
    1. 코틀린은 어느 특정 프로그래밍 스타일이나 패러다임을 사용할것을 강제로 요구하지 않는다.
    2. 인텔리J 아이디어의 개발과 컴파일러의 개발이 맞물려 이뤄져 왔다.
    3. 여러 특성을 배울때도 IDE의 코틀린 언어 지원이 중요한 역할을 한다.
    
    - 간결성
    
    1. 코드가 간단하고 간결할수록 내용을 파악하기가 더 쉽다.
    2. 코틀린을 만들면서 프로그래머가 작성하는 코드에서 의미가 없는 부분을 줄이고,
       언어가 요구하는 구조를 만족시키기 위해 별 뜻은 없지만 프로그램에 꼭 넣어야하는 부수적인 요소를 줄이기 위해 노력을 기울였다.
    3. 코틀린은 람다를 지원하기 때문에 작은 코드 블록을 라이브러리 함수에 쉽게 전달할 수 있다.
    
   
    - 안전성
    
    1. JVM에서 실행되는 정적 타입 지정 언어로서 코틀린은 다양한 무제를 예방할 수 있다.
    2. 코틀린의 타입 시스템은 null이 될수 없는 값을 축적하며, 실행 시점에 NullPointerExcepion이 발생할 수있는 연산을 사용하는 코틀를 금지한다.
    
    - 상호운용성
    
    1. 자바 메서드를 호출하거나 자바 클래스를 상속하거나 인터페이스를 구현하거나 자바 애노테이션을 코틀린 코드에 적용하는 등의 일이 모두 가능하다.
    2. 제공하는 도구도 다중 언어 프로젝트를 완전히 지원한다.
    3. 자바와 코틀린 소스 파일을 자유롭게 내비게이션 할수 있다.
    4. 여러 언어로 이뤄진 프로젝트를 디버깅하고 서로 다른 언어로 작성된 코드를 언어와 관계없이 한단계씩 실행할 수 있다.
    5. 자바 메서드를 리팩토링해도 그 메서드와 관련 있는 코틀린 코드까지 제대로 변경된다.
~~~
